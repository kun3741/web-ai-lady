# Додаток до ТЗ: мульти-модельна робота

## Мульти-модельна робота (важлива правка)

Бот повинен бути універсальним і підтримувати роботу з кількома дівчатами / Telegram-акаунтами.

Стиль спілкування, логіка поведінки, обробка заперечень та загальні правила залишаються однаковими: використовується єдиний базовий стиль власниці / адміністратора.

При підключенні бота до нового акаунту повинна бути можливість легко змінити тільки персональні дані конкретної дівчини:

- ім'я та біографію дівчини;
- номер телефону;
- WhatsApp, тільки для дзвінків;
- реквізити для оплат;
- папку з контентом: фото, відео та відеокружочки.

Бот повинен автоматично розуміти, в якому профілі / акаунті він зараз працює, і використовувати відповідні персональні дані та медіа саме для цього акаунту.

При створенні нового кандидата або обробці нового чату кандидат обов'язково прив'язується до конкретного профілю дівчини. Усі повідомлення, чернетки, пам'ять діалогу, медіа, етапи воронки та налаштування автоматизації мають зберігатися і використовуватися з прив'язкою до цього профілю, щоб дані різних моделей не змішувалися.

У Telegram-інтерфейсі має бути зрозумілий розділ керування профілями / акаунтами, де адміністратор може:

- створити новий профіль дівчини;
- вибрати активний профіль;
- змінити персональні дані профілю;
- прив'язати лідів / кандидатів до потрібного профілю;
- переглянути медіа та ліди окремо для кожного профілю.

Очікувана логіка роботи: при підключенні бота до іншого акаунту адміністратор створює або вибирає відповідний профіль дівчини, заповнює її персональні дані та вказує папку / тег медіа. Після цього бот бере ці дані автоматично для всіх діалогів, пов'язаних із цим профілем.
