# Додаток до ТЗ: inline-керування та навчання AI на чатах

## Керування ботом через inline-кнопки

Усі звичайні сценарії керування ботом повинні виконуватися через inline-кнопки в Telegram-інтерфейсі.

Команди залишаються тільки для входу та аварійних дій:

- `/start` - відкрити головне меню;
- `/panic` - екстрено поставити автопілот на паузу;
- `/clear_all` - повна очистка бази з додатковим підтвердженням.

Звичайні дії не повинні вимагати введення команд вручну:

- створення та вибір профілю дівчини;
- редагування біографії, телефону, WhatsApp, реквізитів і папки контенту;
- створення та перегляд лідів;
- зміна статусу ліда;
- зміна етапу воронки;
- зміна режиму автоматизації;
- генерація, переписування, підтвердження або відхилення чернеток;
- перегляд статистики;
- запуск імпорту та перегляд статусу імпорту;
- керування медіа.

## Навчання AI на історії чатів

Під "навчанням" для цього бота на першому етапі мається на увазі не обов'язково fine-tuning моделі, а створення якісного AI-профілю на основі реальних чатів.

Рекомендований процес:

1. Імпортувати багато Telegram-чатів у форматі JSON.
2. Для кожного чату визначити, хто є дівчиною / моделлю, а хто кандидатом.
3. Витягнути пари повідомлень: "кандидат написав" -> "дівчина відповіла".
4. Очистити дані від дублікатів, службових повідомлень, дуже коротких або неінформативних реплік.
5. Розмітити приклади тегами: знайомство, small talk, турбота, дзвінок, зустріч, подорож, заперечення, конфлікт, пауза тощо.
6. Зберегти найкращі приклади як few-shot style examples у профілі конкретної дівчини.
7. Окремо зберігати стабільні факти в пам'ять: біографія, звички, заборонені теми, типові формулювання, межі спілкування.
8. Перед генерацією відповіді збирати контекст: профіль дівчини, етап воронки, останні повідомлення, пам'ять по кандидату, релевантні приклади стилю.
9. Періодично перевіряти якість відповідей на тестових чатах і додавати невдалі кейси як правила або нові приклади.

Fine-tuning варто розглядати пізніше, коли буде достатньо очищених і стабільно розмічених прикладів. Для старту безпечніше і гнучкіше використовувати комбінацію:

- системний промпт із загальними правилами;
- персональні дані профілю;
- few-shot приклади з реальних чатів;
- пам'ять по конкретному кандидату;
- правила безпеки та ручне підтвердження чутливих тем.
